3. What I Changed From AI Text

Solutions generated by AI were rarely copied word-for-word.

Common modifications were:

Matching TypeScript types to existing interfaces in the project.
Replace generic examples with project-specific API payloads.
Adjusting the React Query examples to match the current query key structure.
How to add suggestions to current Zustand store architecture.
Refactored the generated code to the component organization of the project.
Simplifying solutions where AI has recommended unnecessary abstraction.
Refactoring UI code to match the design that already exists in the project.

Most of the responses were more guidance on implementation than final code.

4. What AI Overlooked or went wrong

Some AI suggestions that had to be corrected:

Wrong assumptions on type

Sometimes the AI would treat the API response as a single object, not an array, causing mismatches in the React Query generics.

Partial Integration Recommendations

Some examples worked in isolation but did not take the existing Zustand store into account and required some extra work to glue together.

Problems with query invalidation

Some of the React Query suggestions correctly invalidated queries, but they didn't update the UI immediately because the current component structure needed explicit refetching.

Edge Cases In Form Initialisation

Some generated logic did not handle undefined form configurations correctly on the initial render. This caused runtime errors that required manual debugging.

Overengineering

A few solutions introduced additional abstraction that was unnecessary for the scope of this assignment and were simplified before implementation.

Reflection

AI accelerated development by helping with troubleshooting, architecture decisions, and implementation ideas. However, every non-trivial feature still required manual reasoning, debugging, integration, and testing.

The most valuable use of AI during this project was as a debugging and design assistant rather than as a source of production-ready code.
